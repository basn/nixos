# Hermes VM

Hermes runs as native NixOS services. Docker is used only by the terminal
backend to isolate commands generated by the agent.

## VM definition

Create the storage dataset and blank disk on `skullcanyon`:

```sh
sudo zfs create \
  -o mountpoint=legacy \
  -o compression=zstd \
  -o atime=off \
  osdisk/vms/hermes
sudo qemu-img create -f raw /var/lib/libvirt/images/hermes/hermes.raw 120G
```

Build or download a NixOS installer ISO, then create the VM:

```sh
nix build .#nixosConfigurations.minimalIso.config.system.build.isoImage

sudo virt-install \
  --name hermes \
  --memory 12288 \
  --vcpus 4 \
  --cpu host-model \
  --machine q35 \
  --boot uefi \
  --disk path=/var/lib/libvirt/images/hermes/hermes.raw,format=raw,bus=virtio \
  --cdrom result/iso/nixos-minimal-*.iso \
  --network bridge=br7,model=virtio,mac=52:54:00:48:45:52 \
  --graphics spice \
  --video virtio \
  --channel unix,target.type=virtio,target.name=org.qemu.guest_agent.0 \
  --autostart \
  --noautoconsole
```

Install NixOS from the console using:

- GPT
- 1 GiB EFI system partition labelled `boot`
- Remaining space in ZFS pool `osdisk`
- Datasets `root`, `nix`, `var`, `home`, and `hermes`
- `compression=zstd`, `atime=off`, and no deduplication

Clone this repository in the installer, mount the datasets under `/mnt`, and
install the declared configuration:

```sh
sudo nixos-install --flake git+https://codeberg.org/basn/nixos#hermes
```

## Required interactive bootstrap

Create an Authentik public OAuth2/OIDC provider:

- Client ID: `hermes-dashboard`
- Issuer: `https://auth.basn.se/application/o/hermes/`
- Redirect URI: `https://hermes.basn.se/auth/callback`
- Authorization code flow with PKCE S256
- Scopes: `openid profile email`

Authenticate the system Hermes instance to the ChatGPT Codex provider:

```sh
sudo -u hermes -H env \
  HOME=/var/lib/hermes \
  HERMES_HOME=/var/lib/hermes/.hermes \
  hermes auth add codex-oauth
```

Then restart the gateway:

```sh
sudo systemctl restart hermes-agent hermes-dashboard
```

## Checks

```sh
systemctl status hermes-agent hermes-dashboard docker
sudo -u hermes -H env HOME=/var/lib/hermes HERMES_HOME=/var/lib/hermes/.hermes hermes doctor
curl -s http://127.0.0.1:9119/api/status | jq
docker ps --filter label=hermes-agent=1
zfs list osdisk/hermes
zfs list -t snapshot -r osdisk/hermes
```
